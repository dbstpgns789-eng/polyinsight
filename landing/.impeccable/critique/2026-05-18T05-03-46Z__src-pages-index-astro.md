## 5. 우선순위 이슈 (Priority Issues)

### [P1] 모든 CTA 링크가 플레이스홀더 (`href="#"`)

**What**: 히어로 CTA, 내비게이션 CTA, 하단 CTA — 7개 버튼 전부 `href="#"`. 클릭하면 페이지 맨 위로 스크롤된다.

**Why it matters**: "무료로 시작하기"를 클릭하는 사용자는 회원가입 페이지 또는 실제 제품으로 이동할 것을 기대한다. 클릭 후 페이지 맨 위로 이동하면 제품이 작동하지 않는다는 신호를 준다. 신뢰 기반 제품에서 이것은 치명적.

**Fix**: 실제 앱 URL 또는 대기자 명단 페이지로 연결. 최소한 `#signup` 앵커를 만들어 인라인 이메일 수집 폼으로 이동하게 할 것.

**Suggested command**: `/impeccable harden` (CTA 연결, 상태 피드백 추가)

---

### [P2] eyebrow 레이블 5개 섹션 전체 반복

**What**: `.eyebrow` 클래스가 히어로(무료 베타 운영 중), 작동 방식, 주요 기능, 대상, 정확성 — 모든 섹션에 반복 적용.

**Why it matters**: `brand.md` 절대 금지 항목: *"반복되는 작은 대문자 트래킹 레이블이 모든 섹션 제목 위에... AI 스캐폴딩"*. 이 패턴은 자동 생성 의심을 높이고, 강조를 소진시킨다.

**Fix**: eyebrow를 최대 2곳으로 제한. 히어로("무료 베타 운영 중")는 유지 — 실제로 유용한 정보. 나머지 섹션은 eyebrow 없이 h2 제목만 두어라. 제목이 충분히 강하면 레이블이 필요 없다.

**Suggested command**: `/impeccable clarify` (섹션 헤딩 카피 정비)

---

### [P2] 모바일에서 카드 비주얼이 텍스트보다 먼저 렌더링

**What**: `@media (max-width: 900px)`에서 `.hero__visual { order: -1; }` 적용. 모바일 사용자는 카드 목업 UI를 먼저 보고, 그 다음에 "논문 PDF 하나로, 카드뉴스 완성" 헤드라인을 본다.

**Why it matters**: 제품 맥락 없이 UI를 먼저 보면 "이게 뭔가?" 반응을 유발한다. Jordan(첫 방문자) 페르소나가 5초 안에 제품을 이해하는 데 장애가 된다.

**Fix**: `order: -1` 제거. 모바일에서는 텍스트 먼저, 비주얼 두 번째 순서 유지. 또는 모바일에서 카드 스택을 `display: none`으로 숨기고 섹션 아래쪽에 간소화된 버전을 표시.

**Suggested command**: `/impeccable adapt` (모바일 레이아웃 최적화)

---

### [P2] `<skip-to-main>` 링크 없음

**What**: 페이지 최상단에 키보드 사용자를 위한 "본문으로 건너뛰기" 링크가 없다.

**Why it matters**: 키보드 또는 스크린 리더 사용자는 TAB 키를 누를 때마다 스티키 네비게이션의 모든 링크(로고, 작동 방식, 주요 기능, 요금제, 로그인, CTA)를 거쳐야 메인 콘텐츠에 도달한다. WCAG 2.4.1 (A 수준) 위반.

**Fix**: `<main>` 앞에 skip 링크 추가:
```html
<a href="#main-content" class="skip-link">본문으로 건너뛰기</a>
```
```css
.skip-link {
  position: absolute;
  transform: translateY(-100%);
  padding: 8px 16px;
  background: var(--accent);
  color: white;
}
.skip-link:focus { transform: translateY(0); }
```

**Suggested command**: `/impeccable audit` (접근성 전체 검사)

---

### [P3] `color-mix(in oklch, ...)` 폴백 없음

**What**: 내비게이션 배경에 `color-mix(in oklch, var(--bg) 82%, transparent)` 사용. 이 CSS 함수는 Chrome 111+, Firefox 113+, Safari 16.2+ 이후 지원.

**Why it matters**: 구형 브라우저에서 내비게이션 배경이 완전 불투명(`var(--bg)`)으로 렌더링되며, 블러 효과 없이 표시된다. 기능은 작동하지만 의도한 frosted-glass 효과가 사라진다.

**Fix**: 폴백 지정:
```css
.nav {
  background: var(--bg); /* fallback */
  background: color-mix(in oklch, var(--bg) 82%, transparent);
}
```

## 7. 소소한 관찰 (Minor Observations)

- **`<ol class="steps" role="list">`**: `<ol>` 자체가 이미 ordered list이므로 `role="list"`는 중복. 제거해도 무방.
- **히어로 서브텍스트 줄바꿈**: `hero__sub`의 두 문장이 합쳐져 있음. 문장 분리가 더 명쾌한 인상을 줄 수 있음.
- **피처 05번이 `reveal-delay-2`를 03번과 공유**: 같은 딜레이를 공유하면 그리드의 다른 행에서 같은 타이밍에 애니메이션됨. 의도적 동기화이면 좋지만 비직관적일 수 있음.
- **풋터 링크 전부 `"#"`**: 개인정보처리방침, 이용약관, 문의하기 — 세 링크 모두 플레이스홀더.
- **`<section>` aria-labelledby**: 모든 섹션에 `aria-labelledby`가 정확히 연결됨. 접근성 구현 우수. ✓
- **`font-variant-numeric: tabular-nums`**: 수치 표시에서 정확히 적용됨. 학술 도구로서 적절한 세부 구현. ✓
