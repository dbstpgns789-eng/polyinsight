# 내보내기 모달 — 콘텐츠 명세

> 카드 에디터 하단바 "카드뉴스 내보내기" 클릭 시 진입.
> CRITICAL 건수가 0일 때만 버튼 활성화됨. HIGH는 경고만 표시하며 진입 허용.
> 상태 4개: PREFLIGHT → RENDERING → DONE | ERROR

---

## 모달 헤더 (공통)

- 제목: `카드뉴스 내보내기`
- 닫기 버튼: RENDERING 상태에서는 없음, 나머지 상태에서는 있음

---

## 상태 1 — PREFLIGHT (내보내기 전 확인)

### 섹션 레이블
- `내보내기 전 확인`

### 체크리스트 (자동 확인, 수동 조작 없음)
1. `총 {N}장의 카드 준비 완료`
2. `이미지 슬롯 {설정된 수}/{전체}장 설정`
   - 전체 설정 시: 그대로 표시
   - 일부 미설정 시: `이미지 슬롯 {설정된 수}/{전체}장 설정 (일부 없음 — 내보내기 가능)`
3. `CRITICAL 필드 검토 완료`
   - HIGH 건수가 있을 때: `HIGH {N}건 — 내보내기는 가능하나 검토 권장` (경고, 차단 아님)

### 실행 버튼
- 텍스트: `내보내기 시작`

---

## 상태 2 — RENDERING (렌더링 중)

- 안내: `PNG 렌더링 중`
- 보조 안내: `Playwright로 카드를 생성하고 있습니다.`
- 닫기 불가

---

## 상태 3 — DONE (완료)

- 완료 제목: `내보내기 완료`
- 보조 안내: `{N}장의 PNG가 준비됐습니다.`
- 버튼: `ZIP 전체 다운로드`
- 카드별 개별 다운로드: `01`, `02`, `03` … (카드 번호별 PNG 파일)

---

## 상태 4 — ERROR (실패)

- 오류 제목: `내보내기 실패`
- 오류 상세: `{서버 에러 메시지}`
- 버튼: `다시 시도` (PREFLIGHT로 복귀)

---

## 제약 조건

| 항목 | 값 |
|------|----|
| 진입 조건 | CRITICAL 건수 0일 때만 가능 (HIGH는 경고만) |
| 이미지 없는 카드 | 내보내기 허용 (경고만 표시) |
| 렌더링 중 닫기 | 불가 |
| 다운로드 형식 | ZIP (전체) / PNG (카드별 개별) |
