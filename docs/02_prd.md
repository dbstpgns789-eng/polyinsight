# Product Requirements Document (PRD)
> PolyInsight v2.0 | 2025-05-05

---

## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| **제품명** | PolyInsight |
| **목적** | 논문 PDF → KITECH 웹진 카드뉴스 자동 생성 |
| **대상** | KITECH 연구자 및 홍보팀 |
| **형태** | 웹 애플리케이션 (SPA, Single Page Application) |
| **현재 버전** | v2.0 (MVP) |

### 왜 이 제품이 필요한가

KITECH 연구자는 논문 발표 후 웹진용 카드뉴스를 디자인팀에 외주한다.
현재 프로세스는 1~2주 소요, 수차례 피드백 반복이 필수다.
PolyInsight는 AI가 논문을 읽고 초안을 자동 생성하여 이 병목을 **1시간 이내**로 단축한다.

> 자세한 문제 정의 → [01_problem_definition.md](./01_problem_definition.md)

## 2. 핵심 사용자 흐름 (Happy Path)

```
1. PDF 업로드       드래그앤드롭 또는 파일 선택
        ↓
2. AI 파이프라인    백그라운드 자동 실행 (~60초)
   S1 텍스트 추출
   S2 섹션 파싱
   S3 한국어 요약  ─┐ 병렬
   S4 핵심 기여   ─┘
   S6 카드 JSON 생성
        ↓
3. 카드 에디터 진입  AI 초안 확인 (3패널: 콘텐츠 | 프리뷰 | 디자인)
        ↓
4. 사람 검토        수치 검증 + 이미지 업로드 + 레이아웃 선택
        ↓
5. PNG 내보내기      Playwright 렌더링 → 5장 × 1080×1080 → ZIP 다운로드
```

**설계 원칙**: AI는 초안만 생성한다. 수치·사실의 최종 책임은 사람에게 있다.

## 3. 기능 요구사항

### FR-UPLOAD — 업로드

| ID | 요구사항 |
|---|---|
| FR-UP-001 | PDF 파일만 허용. 최대 50 MB. 다른 형식은 업로드 전 차단. |
| FR-UP-002 | 텍스트 레이어 없는 PDF(스캔본) 처리 불가. S1 실패 시 명확한 안내 메시지. |
| FR-UP-003 | 업로드 후 파이프라인 백그라운드 실행. 모달 닫아도 처리 계속. |

### FR-PIPELINE — AI 파이프라인

| ID | 요구사항 |
|---|---|
| FR-PP-001 | S1 텍스트 추출 실패 시 파이프라인 중단. 재시도 버튼 제공. |
| FR-PP-002 | S3(한국어 요약)과 S4(핵심 기여) 병렬 실행 (`asyncio.gather`). |
| FR-PP-003 | S5 단계 없음 — KITECH 버전에서 제거됨. |
| FR-PP-004 | S6는 원문 섹션맵을 1차 근거로 사용. S3/S4는 힌트 전용. 충돌 시 원문 우선. |
| FR-PP-005 | 각 스테이지 완료 시 프론트에 실시간 진행 상태 전달 (폴링 또는 SSE). |
| FR-PP-006 | 스테이지 실패 시 해당 지점부터 재시도 가능. 전체 재실행 불필요. |

### FR-EDITOR — 카드 에디터

| ID | 요구사항 |
|---|---|
| FR-ED-001 | 3패널 레이아웃: 왼쪽(콘텐츠 편집) / 중앙(카드 프리뷰) / 오른쪽(디자인 설정). |
| FR-ED-002 | 각 필드에 AI 신뢰도 배지 표시: CRITICAL / HIGH / MEDIUM / LOW. |
| FR-ED-003 | CRITICAL 필드는 원문 출처(섹션명, 페이지 번호) 함께 표시. |
| FR-ED-004 | 카드별 레이아웃 변형 A/B/C/D 선택 가능. AI 추천 변형 별도 표시. |
| FR-ED-005 | 이미지 슬롯 제공. 변형 선택에 따라 슬롯 위치 자동 변경. 이미지 없어도 내보내기 허용. |
| FR-ED-006 | 5초 idle 후 자동저장. 저장 상태 상단 표시 (저장됨 / 저장 중). |
| FR-ED-007 | 하단 ActionBar: CRITICAL/HIGH 건수 0일 때만 내보내기 버튼 활성화. |

### FR-EXPORT — 내보내기

| ID | 요구사항 |
|---|---|
| FR-EX-001 | Playwright 헤드리스 브라우저 렌더링 → 1080×1080 PNG × 5장. |
| FR-EX-002 | 카드 1→5 순차 렌더링. 각 카드 완료 시 모달에 실시간 표시. |
| FR-EX-003 | 전체 ZIP 다운로드 + 개별 카드 PNG 다운로드 모두 지원. |
| FR-EX-004 | 렌더링 실패 시 실패한 카드만 재시도. 완료된 카드 재렌더링 없음. |
| FR-EX-005 | 파일명 형식: `kitech_{slug}_{YYYYMM}.zip` |

### FR-DASHBOARD — 대시보드

| ID | 요구사항 |
|---|---|
| FR-DB-001 | 프로젝트 목록. 상태별 필터: 완성 / 임시저장 / 처리중 / 실패. |
| FR-DB-002 | 프로젝트 카드에 5장 미니 썸네일 스트립 표시. |
| FR-DB-003 | 처리중 프로젝트에 실시간 파이프라인 진행률 표시. |
| FR-DB-004 | 실패 프로젝트에 재시도 버튼 제공. |

### FR-PROFILE — 기관 프로필

| ID | 요구사항 |
|---|---|
| FR-PR-001 | 기관명, 캐릭터 이미지, 로고, CTA 문구를 1회 등록. |
| FR-PR-002 | 등록 후 모든 카드뉴스에 자동 주입. 카드별 재입력 불필요. |
| FR-PR-003 | 연구자 사진 라이브러리 관리. 연구자 이름 매칭으로 자동 주입. |

## 4. 비기능 요구사항

| 항목 | 목표 | 비고 |
|---|---|---|
| 파이프라인 처리 시간 | 60초 이내 (S1~S6) | LLM API latency 포함 |
| Playwright 렌더링 | 5장 25초 이내 (카드당 최대 5초) | 서버 사양에 따라 변동 |
| 동시 사용자 | MVP: 5명 | 큐 없음, 선착순 처리 |
| 데이터 보존 | SQLite 영구 저장. 생성 파일(PNG/ZIP) 24시간 TTL 후 삭제. | |
| 가용성 | MVP: 단일 프로세스 허용. 다운타임 SLA 없음. | |
| 보안 | MVP: 인증 없음. 내부망 전용 배포 전제. | |

## 5. 범위 밖 (MVP 제외)

아래 기능은 현재 버전에서 구현하지 않는다.
요청이 들어오더라도 scope 외로 처리한다.

| 제외 항목 | 이유 |
|---|---|
| 사용자 계정 / 로그인 | 내부망 전용 MVP. 인증 복잡도 대비 효과 낮음. |
| SNS 자동 게시 | 승인 프로세스 미정. 외부 API 연동 공수 과다. |
| OCR (스캔 PDF) | 텍스트 품질 불안정. 별도 처리 파이프라인 필요. |
| 복수 논문 비교 | 현재 단일 논문 → 단일 카드뉴스 흐름만 정의됨. |
| AI 이미지 자동 생성 | 생성 이미지의 공신력 문제. 기관 홍보물에 부적합. |
| 다국어 출력 | 현재 한국어 출력만 정의됨. |

## 6. 카드뉴스 품질 기준

카드뉴스는 공식 기관 홍보물이다. 아래 기준을 모두 충족해야 출판 가능으로 판단한다.

| 기준 | 설명 |
|---|---|
| **수치 정확성** | 모든 수치는 논문 원문에 존재해야 함. AI가 추론한 수치 사용 금지. |
| **사실 범위** | 논문에 없는 내용 추가 금지. 과장 또는 비교 우위 표현 금지. |
| **언어 수준** | 전문용어 최소화. 일반인이 30초 내 핵심 이해 가능해야 함. |
| **브랜드 준수** | KITECH 브랜드 컬러 — Navy `#1B2F7A`, Yellow `#F5C400`. 로고 위치 고정. |
| **연구자 동의** | 연구자가 "내 연구가 제대로 표현됐다"고 동의하는 수준. |

## 8. 의존성 및 기술 스택

| 레이어 | 기술 | 선택 이유 |
|---|---|---|
| Frontend | React + Vite + Tailwind CSS | 빠른 개발, SPA |
| Backend | FastAPI (Python) | async 지원, LLM 연동 용이 |
| LLM | Claude API (Anthropic) | 한국어 품질, 긴 문서 처리 |
| PDF 파싱 | pdfplumber / PyMuPDF | 텍스트 레이어 추출 |
| 렌더링 | Playwright (headless Chromium) | 픽셀 정확도, CSS 완전 지원 |
| 저장소 | SQLite | MVP 단순성. 추후 PostgreSQL 전환 가능. |
