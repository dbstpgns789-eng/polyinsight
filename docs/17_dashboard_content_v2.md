# 17 · 대시보드 콘텐츠 요구사항
> v2.0 | 2026-05-20
> 화면 구조·레이아웃·컴포넌트 표현 없음.
> "이 화면에는 이런 내용이 있어야 한다"만 정의.

---

## 이 화면의 목적

로그인한 사용자가 자신의 카드뉴스 작업 현황을 파악하고,
기존 작업을 이어가거나 새 작업을 시작하는 공간.

---

## 1. 사용자가 이 화면에서 알아야 할 것

### 1-1. 나의 전체 작업 현황

사용자는 자신이 지금까지 만든 카드뉴스가 얼마나 있는지,
그 중 몇 개가 완성됐고 몇 개가 아직 진행 중인지 한눈에 파악할 수 있어야 한다.

집계가 필요한 상태:
- 전체
- 완성
- 임시저장
- 처리중

### 1-2. 각 프로젝트의 정보

각 카드뉴스 프로젝트에 대해 사용자가 알아야 하는 것:
- 어떤 논문으로 만들었는가 (논문 제목 또는 파일명)
- 언제 마지막으로 작업했는가
- 몇 장짜리 카드뉴스인가
- 지금 이 프로젝트가 어떤 상태인가

### 1-3. 처리 중인 프로젝트의 진행 상황

파이프라인이 돌아가고 있는 프로젝트라면,
지금 어느 단계가 진행 중인지 알 수 있어야 한다.

단계별로 사용자에게 보여줄 문구:
- 텍스트 추출 중...
- 섹션 분류 중...
- 핵심 내용 분석 중...
- 카드 생성 중...
- 저장 중...

### 1-4. 실패한 프로젝트의 실패 이유

실패한 프로젝트라면, 왜 실패했는지 알 수 있어야 한다.

| 실패 원인 | 사용자에게 전달할 메시지 |
|---|---|
| 텍스트를 추출할 수 없는 PDF | 스캔 PDF는 처리할 수 없습니다. |
| 문서 구조 파악 실패 | 문서 구조를 파악하지 못했습니다. |
| 핵심 내용 추출 실패 | 핵심 내용을 추출하지 못했습니다. |
| 카드 생성 실패 | 카드를 생성하지 못했습니다. |

---

## 2. 사용자가 이 화면에서 할 수 있어야 하는 것

### 2-1. 새 카드뉴스 만들기

가장 중요한 행동. 화면 어디서든 접근 가능해야 한다.
레이블: **새 카드뉴스 만들기** (또는 짧게 **새 카드뉴스 +**)

### 2-2. 특정 상태의 프로젝트만 보기

완성/임시저장/처리중/실패/전체 중 원하는 상태만 골라 볼 수 있어야 한다.

### 2-3. 프로젝트 목록 정렬

기준:
- 최신순 (기본값)
- 오래된 순
- 이름순

### 2-4. 완성된 프로젝트 수정

레이블: **수정하기**
동작: 카드 에디터 진입

### 2-5. 완성된 프로젝트 다운로드

레이블: **다운로드**
동작: 에디터 진입 후 내보내기 자동 시작

### 2-6. 임시저장된 프로젝트 이어하기

레이블: **이어하기**
동작: 카드 에디터 진입

### 2-7. 실패한 프로젝트 재시도

레이블: **재시도**
동작: 실패 지점부터 파이프라인 재실행

### 2-8. 로그아웃

레이블: **로그아웃**
동작: 세션 종료 후 랜딩 페이지 이동. 확인 절차 없음.

---

## 3. 계정 관련 표시

사용자가 현재 누구로 로그인돼 있는지 알 수 있어야 한다.
로그인된 이메일 주소를 어딘가에 표시한다.

---

## 4. 빈 상태 — 프로젝트가 하나도 없을 때

처음 가입하거나 프로젝트가 없는 경우.
숫자 요약(현황 집계)은 표시할 필요 없다.

| 요소 | 텍스트 |
|---|---|
| 안내 제목 | 첫 논문을 업로드하세요 |
| 안내 설명 | PDF를 올리면 카드뉴스를 자동으로 만들어 드립니다. |
| 행동 유도 | 새 카드뉴스 만들기 |

---

## 5. 필터 결과가 없을 때

특정 상태를 골랐는데 해당 프로젝트가 없는 경우.

| 필터 | 안내 문구 |
|---|---|
| 완성 | 완성된 카드뉴스가 없습니다. |
| 임시저장 | 임시저장된 프로젝트가 없습니다. |
| 처리중 | 처리 중인 프로젝트가 없습니다. |
| 실패 | 실패한 프로젝트가 없습니다. |
