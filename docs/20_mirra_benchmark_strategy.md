## 1. 포지셔닝 결정 (Decision)

> **PolyInsight = 학술논문판 Mirra.**

Mirra는 일반 마케터용으로 **편집 가능한 AI 카드뉴스 워크플로 + 저장률 공학 + 브랜드 일관성**이라는 플레이북을 이미 증명했다.
우리는 그 플레이북을 **그대로 벤치마킹**하고, 입력을 **논문 원문**으로 좁혀 **fidelity(근거 보증)라는 해자**를 얹는다.
"검증된 수평 도구의 수직 클론"은 강한 스타트업 전략이다.

**"전면 벤치마킹"의 경계 (드리프트 방지):**
- ✅ **베낀다(플레이북):** 편집가능 워크플로, 후수정 UX, 저장률 공학, 브랜드(skin) 일관성, 속도, 발행 전 체크리스트.
- 🚫 **그냥 클론은 안 됨(우리만의 것):** grounding/인용 검증, 학술 톤, 방법론·수치·출처 컴포넌트, 학술 출력 포맷(카드뉴스+연구요약+보도자료+발표자료). 이게 빠지면 "못난 Mirra"가 됨.

## 2. 편집 가능 워크플로 (Dylan 5단계) → 우리 파이프라인 매핑

핵심 명제(Dylan): **"AI가 만들어주느냐"가 아니라 "AI 결과물을 내가 원하는 수준까지 쉽게 고칠 수 있느냐"가 기준.**

| Dylan 워크플로 | PolyInsight 현재 | 상태 |
|---|---|---|
| ① 원본 자료 투입(블로그·URL·PDF·브리프) | S1 PDF 텍스트 추출 | ✅ (논문 PDF) |
| ② AI가 구조 잡기(훅·순서·핵심메시지·CTA) | S6 storyboard(beat·narrative_role·key_message) | ✅ 구조 有, 훅·밀도 규율 ⚠️ |
| ③ 디자인 초안(제목/본문/예시/CTA 한 톤) | S6 필드 + skin/skeleton 한 세트 | ✅ 한 톤 보장(skin) |
| ④ **직접 후수정**(폰트·문구·이미지·여백·강조) | 에디터: 텍스트 인라인 + 요소 미세조정 + 덱 색/글꼴 + 이미지 focal/fit | ✅ 이번에 대거 구축 |
| ⑤ 플랫폼별 발행(IG·블로그·틱톡 포토모드) | export(PNG) — 멀티포맷은 후속 | ⚠️ 단일 포맷 |

**시사점:** 우리 S1-S8이 이미 이 워크플로와 구조적으로 동형. **④ 후수정이 제품 가치의 핵심**이라는 Dylan의 명제가 우리가 이번에 control(요소/색/글꼴)을 집중 구축한 방향을 검증한다(메모리 [[project-editor-control-direction]]). 갭 = ⑤ 멀티포맷.

## 3. 발행 전 체크리스트 (Dylan) → 우리 에디터/프리플라이트로

| 항목 | 체크 질문 | 우리 적용 |
|---|---|---|
| 첫 장 | 타겟이 멈출 문장인가? | cover hook을 저장유도형으로(→ `19` §2-1). export preflight 경고 후보 |
| 흐름 | 슬라이드마다 한 메시지인가? | 장당 밀도 상한(1메시지·30~50자, → `19` §2-2). **최우선 갭** |
| 디자인 | 브랜드 톤 유지되나? | skin 한 세트 + font_pairing이 구조로 강제 ✅ |
| CTA | 다음 행동 명확한가? | closing 카드(단, `19`는 CTA보다 **요약**이 저장률↑ — 우리 맥락에선 요약+캡션 CTA) |

→ 이 체크리스트를 **에디터 발행 전(export preflight) 경고 항목**으로 코드화 가능. (현 preflight는 미검토/CRITICAL만 경고 — 가독성 항목 추가 여지.)

## 4. 후수정(editability)이 왜 핵심인가 (Dylan 논거)

- 카드뉴스 = 브랜드의 얼굴. 매번 다른 폰트·톤이면 신뢰 하락 → **브랜드 맥락 유지하며 반복 생산**하는 AI가 필요(속도만 빠른 AI ❌).
- 작은 오타·어색한 표현 때문에 **전체를 재생성할 필요가 없어야** 함 → 실제 작업시간 절감. (= 부분 편집 가능성.)
- 바로 발행 ❌ — 실무는 문구·CTA·브랜드톤 **한 번 검수**가 안전.
- 첫 테스트 자료 = 이미 메시지가 정리된 것(발행한 블로그·상세페이지). **우리 버전 = 이미 출판된 논문**(메시지 확정·근거 명확) → 테스트 소재로 이상적.

**검증:** "AI 완성"이 아니라 "AI 초벌 + 사용자 최소 수정 발행"이라는 우리 정체성([[project-editor-control-direction]])과 정확히 일치. 부분 편집(요소 단위 override)을 만든 것도 "전체 재생성 회피" 논거와 맞음.

## 5. 결론 — 다음 우선순위 (벤치마킹 격차 순)

1. **장당 밀도 상한 강제** (S6) — 체크리스트 "흐름" + `19` 최우선 갭. 가독성 직결.
2. **첫 장 훅 카피** 저장유도형(S6).
3. **closing = 요약** 카드.
4. **발행 전 체크리스트 UX**(export preflight에 가독성 4항목).
5. **멀티포맷 출력**(⑤) — 학술 특화: 카드뉴스 + 연구요약 1-pager + 보도자료 + 발표 슬라이드.
6. (해자) **인용 검증 UX** — 숫자 클릭→원문 점프. Mirra엔 없는 학술 전용.
