# 이미지 배치 v1 — 업로드 배선 + 적응형 (cover_v2 · feature)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 사용자가 cover_v2·feature 카드에 이미지를 업로드할 수 있게 배선하고, 두 뼈대를 이미지 유무에 적응시킨다(없으면 텍스트가 채우고 빈 박스가 안 보임). "이미지가 없어서" 병목의 최소 해소 v1.

**Architecture:** 프론트 전용. 이미지 콘텐츠=사용자 권한(업로드만, AI 생성 없음). 뼈대=함수: `(텍스트, 이미지?)`를 받아 적응(적응형 원칙 재사용). 업로드→`image_url`은 이미 제네릭 배선됨(`handleImageUpdate`); RightPanel 업로드 UI는 `getSlotType(template_type) !== 'none'` 게이트라 imageSlots에 신규 키만 추가하면 활성.

**Tech Stack:** Next 16 · React 19 · TS. 기존 `EditableImage`/`VisualZone`/RightPanel 재사용.

**검증:** 브라우저 — `/editor/demo`에서 cover(14)·feature(16) 선택 시 RightPanel 이미지 업로드 뜨고, 올리면 카드에 반영, 이미지 없는 상태에선 빈 틴트 박스 대신 텍스트가 채움.

**비목표(피드백 후):** statement/closing 배경 이미지(①층), 데이터 4종 이미지, figure 자동 추출, 이미지 생성 API, focal point 드래그 UI, 백엔드 IMAGE_SLOT_TYPES 동기화(프론트 우선).

**설계 기준(고정):** 이미지는 "메시지를 돕는 곳"에만. v1 = 기존 VisualZone 보유 뼈대(cover_v2·feature)만. 손으로 조합 배치 안 함 — 뼈대가 적응.

## Task 1: imageSlots — 'zone' 타입 + cover_v2·feature 매핑

**Files:** Modify `web/src/lib/imageSlots.ts`, `web/src/components/editor/RightPanel.tsx`

- [ ] **Step 1: imageSlots.ts**

`SlotType`에 `'zone'` 추가:
```ts
export type SlotType = 'bg' | 'inset_top' | 'inset_right' | 'inner' | 'zone' | 'none'
```
`IMAGE_SLOT_TYPES`에 신규 뼈대 매핑 추가(기존 12키 유지):
```ts
export const IMAGE_SLOT_TYPES: Record<string, SlotType> = {
  cover:      'bg',
  hook:       'bg',
  problem:    'bg',
  circle3:    'bg',
  compare2:   'bg',
  grid4:      'bg',
  flow:       'bg',
  showcase:   'inset_top',
  definition: 'inset_right',
  closing:    'inner',
  data:       'none',
  brand:      'none',
  // 신규 뼈대 (v1: 이미지 받는 둘만 'zone', 나머지는 미등록→'none')
  cover_v2:   'zone',
  feature:    'zone',
}
```
`SLOT_META`에 `'zone'` 항목 추가:
```ts
  zone: {
    type:        'zone',
    label:       '이미지 영역',
    description: '카드의 지정된 이미지 자리에 표시됩니다 (없으면 텍스트가 채웁니다)',
  },
```

- [ ] **Step 2: RightPanel SlotDiagram 확인**

`RightPanel.tsx`를 읽어 `slotMeta.type`을 switch/조건분기하는 SlotDiagram(또는 슬롯 시각화)이 있는지 본다.
- 있고 `'zone'` 케이스가 없으면: 기존 케이스 스타일을 따라 `'zone'` 케이스(간단한 사각 영역 다이어그램 또는 라벨만) 추가.
- `slotMeta.label`/`description`만 텍스트로 쓰는 구조면: 추가 작업 불필요(Step 1의 meta가 자동 표시됨).
업로드 게이트는 `hasSlot = slotMeta.type !== 'none'`이므로 Step 1만으로 cover_v2·feature 업로드가 활성된다.

## Task 2: Cover·Feature 이미지 적응

**Files:** Modify `Cover.tsx`, `Feature.tsx`

이미지 없음 + 비편집 모드 → VisualZone 빈 박스를 렌더하지 않고 텍스트/공간이 채운다. 편집 모드 → VisualZone 표시(업로드 affordance).

- [ ] **Step 1: Feature.tsx — 이미지 컬럼 적응**

`Feature.tsx`에서 본문 영역(`<div style={{ flex: 1, display: 'flex', gap: 40, ... }}>`)의 VisualZone 컬럼을 조건부로. props에 이미 `card`·`mode`·`onImageRequest` 있음. 다음과 같이:

함수 본문 상단(`const { ... } = props` 다음)에 추가:
```tsx
  const showImage = !!card.image_url || mode === 'edit'
```
이미지 컬럼 div를 조건부로 감싼다(기존):
```tsx
        <div style={{ flex: 1, minHeight: 0 }}>
          <VisualZone imageUrl={card.image_url} slotKey="feature" mode={mode} onImageRequest={onImageRequest} />
        </div>
```
→
```tsx
        {showImage && (
          <div style={{ flex: 1, minHeight: 0 }}>
            <VisualZone imageUrl={card.image_url} slotKey="feature" mode={mode} onImageRequest={onImageRequest} />
          </div>
        )}
```
(이미지 없고 render/thumbnail이면 텍스트 컬럼 1개만 남아 flex로 폭 전체를 자연히 채운다.)

- [ ] **Step 2: Cover.tsx — VisualZone 밴드 적응**

`Cover.tsx`에서 함수 본문 상단에 추가:
```tsx
  const showImage = !!card.image_url || mode === 'edit'
```
VisualZone 밴드(기존):
```tsx
      <div style={{ flex: 1, minHeight: 0, marginTop: 32, marginBottom: 24 }}>
        <VisualZone imageUrl={card.image_url} slotKey="cover" mode={mode} onImageRequest={onImageRequest} />
      </div>
```
→ 이미지 있으면 밴드, 없으면 빈 공간(레이아웃 유지로 caption이 바닥에):
```tsx
      {showImage ? (
        <div style={{ flex: 1, minHeight: 0, marginTop: 32, marginBottom: 24 }}>
          <VisualZone imageUrl={card.image_url} slotKey="cover" mode={mode} onImageRequest={onImageRequest} />
        </div>
      ) : (
        <div style={{ flex: 1 }} />
      )}
```

Run: `cd web; npx tsc --noEmit` → 0 errors.


**Rules:** 두 뼈대만. 강제규칙 유지(색/배경/폰트/border 직접 정의 금지 — 조건부 layout div만 추가). VisualZone/CardSurface 등 피부는 그대로 조립.
