> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 딥틸 세트의 피부 토큰 + 7개 피부 컴포넌트 + ⑤BigStatCompare 뼈대 1개를 만들어, 첫 카드 한 장을 브라우저/PNG에서 끝까지 렌더한다.

**Architecture:** 피부(skin)/뼈대(skeleton) 분리. `--set-*` 토큰을 `CardFrame`이 주입 → 피부 컴포넌트가 토큰을 읽어 스타일 소유 → 뼈대는 피부 컴포넌트만 조립(자기 색·폰트·박스 정의 안 함). 순수 파싱 로직은 React에서 분리한 `parse.ts`에 모아 단위테스트한다. 기존 섬 12개·렌더 라우트·에디터는 건드리지 않는다(후방호환).

**Tech Stack:** Next.js 16 · React 19 · TypeScript · CSS custom properties(`--set-*`) · 기존 `EditableText` contenteditable 프리미티브 · Vitest(순수 로직 단위테스트) · Playwright PNG(기존 S7, 변경 없음).

**검증 방식(두 갈래):**
- **순수 로직(파서/구분자)** → **Vitest 단위 테스트(TDD)**. `parse.ts`·`delimiters.ts`처럼 React 없는 순수 함수는 라운드트립/엣지케이스를 단위테스트로 잡는다. (Task 2에서 Vitest 도입.)
- **시각 컴포넌트(피부/뼈대)** → **브라우저 시각 검증**. 이 저장소는 카드 시각 회귀를 S7 전환 때 접었고(CLAUDE.md §13 "빌드 성공이 완료기준 아님"), 흰박스·색일관·폰트는 단위테스트로 못 잡는다. 시각 컴포넌트 태스크는 `tsc --noEmit` + `npm run lint`로 닫고, 통합(Task 13)에서 브라우저로 최종 확인한다.

이유: 단위테스트는 "라벨에 `:`가 섞이면 조용히 깨지는" 파서 버그엔 강하지만, "안 읽힌다"는 시각 실패엔 무력하다. 비례 원칙으로 로직에만 단위테스트를 건다.

**비목표(후속 플랜):** 섬 12개 철거 · `Card.items[]` 데이터모델 이전 · 나머지 7개 뼈대(Cover·Statement·Feature·Process·Reasons·Grid·Closing) · S7 render 라우트의 skeleton dispatch 교체 · method/review 아키타입 · 데이터-구동 세트 선택 UI.

**스펙 드리프트 노트(구현 중 발견 → 스펙 패치 필요):**

```ts
// 카드 피부의 순수 파싱 로직. React 의존 없음 → 단위테스트 대상.
// Headline(강조)·CompareBars(비교행)·BigStatCompare 뼈대가 여기서 import.

/** "a *b* c" → [{text:'a ',em:false},{text:'b',em:true},{text:' c',em:false}] */
export function parseEmphasis(value: string): Array<{ text: string; em: boolean }> {
  const out: Array<{ text: string; em: boolean }> = []
  const re = /\*([^*]+)\*/g
  let last = 0
  let m: RegExpExecArray | null
  while ((m = re.exec(value)) !== null) {
    if (m.index > last) out.push({ text: value.slice(last, m.index), em: false })
    out.push({ text: m[1], em: true })
    last = m.index + m[0].length
  }
  if (last < value.length) out.push({ text: value.slice(last), em: false })
  return out
}

export interface CompareRow {
  label: string
  value: string      // 숫자 문자열(표시 + 너비 계산)
  primary: boolean
}

/** "a:238:1|b:199:0" → CompareRow[] */
export function parseCompareRows(raw: string | undefined): CompareRow[] {
  if (!raw) return []
  return raw.split('|').map((seg) => {
    const [label = '', value = '', primary = '0'] = seg.split(':')
    return { label: label.trim(), value: value.trim(), primary: primary.trim() === '1' }
  }).filter((r) => r.label.length > 0)
}

/** CompareRow[] → "a:238:1|b:199:0". parseCompareRows의 역. */
export function rowsToRaw(rows: CompareRow[]): string {
  return rows.map((r) => `${r.label}:${r.value}:${r.primary ? '1' : '0'}`).join('|')
}
```

```tsx
import EditableText from '../shared/EditableText'
import { parseEmphasis } from './parse'

interface HeadlineProps {
  value: string
  fieldKey: string
  mode: 'edit' | 'render' | 'thumbnail'
  onFieldChange?: (fieldKey: string, value: string) => void
  onFieldFocus?: (fieldKey: string) => void
  focused?: boolean
}

const headlineStyle = {
  display: 'block',
  fontFamily: 'var(--set-font)',
  fontSize: 'var(--set-headline)',
  fontWeight: 800,
  lineHeight: 1.22,
  letterSpacing: '-0.02em',
  color: 'var(--set-ink-strong)',
  wordBreak: 'keep-all' as const,
}

export default function Headline({ value, fieldKey, mode, onFieldChange, onFieldFocus, focused }: HeadlineProps) {
  if (mode === 'edit') {

        fieldKey={fieldKey}
        value={value}
        mode={mode}
        multiline
        onFieldChange={onFieldChange}
        onFieldFocus={onFieldFocus}
        focused={focused}
        style={headlineStyle}
      />
    )

  return (
    <div style={headlineStyle}>
      {parseEmphasis(value).map((seg, i) =>
        seg.em
          ? <em key={i} style={{ color: 'var(--set-accent)', fontStyle: 'normal' }}>{seg.text}</em>
          : <span key={i}>{seg.text}</span>
      )}
    </div>
  )
}
```
