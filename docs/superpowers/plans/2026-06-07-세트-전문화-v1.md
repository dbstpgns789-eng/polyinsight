# 세트 전문화 v1 — 장식·타이포 어휘 심화 Implementation Plan

> **For agentic workers:** 이 플랜은 **시각 크래프트** 작업이다. 순수 TDD가 아니라 단계마다
> **브라우저 시각 게이트**(스크린샷 → 사람이 판단)로 검증한다. 로직(parse/신규 컴포넌트 구조)은
> 실제 단위테스트로. 실행은 `superpowers:executing-plans`(인라인+체크포인트) 권장 —
> 시각 품질은 오케스트레이터가 눈으로 봐야 하므로 블라인드 서브에이전트보다 인라인이 맞다.
> Steps use checkbox (`- [ ]`) syntax.

**Goal:** 단일 카드 세트를 "정보카드 에디토리얼"(밝은 종이 + 진한 포레스트 + 굵은 타이포) 미감의
전문 제작업체 수준으로 끌어올린다. 색이 아니라 컴포넌트 어휘 깊이로.

**Architecture:** 피부(skin)/뼈대(skeleton) 분리 유지. 토큰(`sets.ts`) + 피부 컴포넌트(스타일 소유) 층만
심화. 뼈대는 토큰·컴포넌트만 조립(강제규칙: 색/배경/폰트/border 0건). 미감 방향 C는 2026-06-07 시각
스파이크로 확정 — 기준 토큰은 `web/src/app/render/spike/sets.spike.ts`의 `C_BOLD_EDITORIAL`.

**Tech Stack:** Next.js 15 + React 19 + TypeScript, CSS-in-JS(inline style + `--set-*` CSS 변수),
OKLCH 색(DESIGN.md 규율: #000/#fff 금지), Playwright(파이썬, 시각 캡처), Vitest(로직).

**핵심 발견:** 앱 크롬(DESIGN.md "Academic Desk")은 밝은 포레스트 그린인데 카드는 딥틸이라 단절돼
있었다. C(light)로 가면 앱↔카드 디자인 언어가 통일된다.
