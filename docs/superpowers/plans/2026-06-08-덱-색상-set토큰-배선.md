# 덱 색상 → --set-* 오버라이드 배선 (플랜 A) Implementation Plan

> 버그픽스 + 아키텍처 정리. 색상 컨트롤이 죽은 `--theme-*`가 아니라 살아있는 `--set-*`를 건드리게 한다.

**Goal:** 배경/강조 색 컨트롤이 실제 카드(8뼈대+피부)에 반영되게 하고, 레거시 `--theme-*` 네임스페이스를 라이브 렌더 경로에서 은퇴시킨다.

**근본원인 (확정):** 색 컨트롤은 `--theme-bg`/`--theme-primary`를 설정하지만, 카드는 `--set-bg-gradient`/`--set-accent`만 보고 그린다(CardSurface.tsx:17, BgMotif/Eyebrow/Headline). 두 네임스페이스 단절 → 컨트롤이 죽은 손잡이. skeletons는 `theme`를 전혀 안 씀(grep 0건) — `theme`/`--theme-*`는 CardFrame에서만 쓰이는 잔재.

**핵심 제약:** 덱 색은 **선택적 오버라이드**. 미설정(undefined)=세트 기본값, 설정 시에만 `--set-*` 덮음. (기본값 `#111111`/파랑을 그대로 덮으면 전 카드가 검정/파랑으로 회귀 → 금지.)

**Tech Stack:** Next.js15/React19/TS, FastAPI/Pydantic.
