## 1. Problem

실독자 #1 잔여 불만 = "이미지 부재". 현재 8뼈대 중 이미지 슬롯을 가진 건
`cover_v2`·`feature` 둘뿐이고, 초점(focal) 제어 UI가 없어 사진이 정사각형 존에
크롭될 때 어느 부분을 보여줄지 사용자가 조정할 수 없다.

`focal{x,y}`는 `VisualZone`이 prop으로 *지원*만 할 뿐, 데이터 모델에 필드가 없고
Cover/Feature조차 값을 넘기지 않아 **end-to-end로 한 번도 배선된 적이 없다.**

## 2. Decision

### 2.1 모델 — A: 디자인된 역할 고정 (현 모델 심화)

이미지는 뼈대 설계의 일부다. 뼈대마다 정해진 자리(존)에만 들어가고, 사용자는
**"올릴까 말까"만 판단**한다. 임의 위치 배치·AI 생성은 하지 않는다(fidelity).

대안 B(보편 배경 레이어 = 모든 카드에 배경 사진)는 **기각**: [[project-readability-failure]]의
"사진 위 글이 안 읽힘 / 흰박스 이질감" 실패를 구조적으로 막 해소했는데, 보편 배경
레이어는 그 문제를 되살린다.

### 2.2 핵심 원칙 — 담긴 존(contained zone)만

이미지는 항상 둥근 프레임 안에 담기고 **텍스트는 절대 사진 위에 겹치지 않는다.**
배경 풀블리드 + 텍스트 오버레이는 금지. → readability 실패 차단.

### 2.3 스코프 — 이미지 자리를 추가할 뼈대

| 뼈대 (키) | 처리 | 비고 |
|---|---|---|
| 표지 `cover_v2` | 존 (기존) | 상단 텍스트 + 하단 밴드 |
| 혁신 `feature` | 존 (기존) | 좌 텍스트 + 우 존 |
| **진술 `statement`** | **존 (신규)** | 텍스트 위 + **하단 이미지 밴드** |
| **마무리 `closing_v2`** | **존 (신규)** | 텍스트 + **하단 중앙 밴드** |
| 과정 `process_v2` | 없음 | 아이콘·화살표가 이미 시각 |
| 성능 `bigstat_compare` | 없음 | 수치 임팩트 = fidelity 정체성 |
| 근거 `reasons` | 없음 | 아이콘칩이 이미 시각 |
| 응용 `grid_v2` | 없음 | 격자 아이콘이 이미 시각 |

데이터 4종은 사진을 넣으면 오히려 어수선해지므로 **깔끔 유지**. 변경 없음.

### 2.4 무이미지 적응 (v1 패턴 재사용)

Cover/Feature의 검증된 패턴을 Statement/Closing에 확장:

```
showImage = !!card.image_url || mode === 'edit'
```

- 이미지 없음 + 최종 렌더 → 텍스트가 카드 전체를 채우는 별도 분기. **빈 박스 0, 흔적 0.**
- 이미지 없음 + 에디터 → `EditableImage`의 점선 "이미지 추가" 드롭존만 표시.
- 이미지 있음 → 둥근 존에 사진(겹침 없음).

### 2.5 focal 클릭 초점 (P2 신규 작업)

제어 수준 = **A: 클릭 초점만** (드래그·줌·채움/전체 토글은 오버스펙으로 제외).

- **데이터**: `Card`에 `focal?: { x: number; y: number }` 추가. 없으면 center(0.5, 0.5).
- **상호작용**: 편집 모드에서 **이미지 본문 클릭 = 초점 설정.**
  클릭 좌표 ÷ 요소 크기 → `{x,y}`(0~1 클램프) → `objectPosition: '{x*100}% {y*100}%'`.
  현재 초점은 작은 십자 표식으로 표시. 사용자가 계속 클릭하며 맞춘다.
- **교체 vs 초점 충돌 해소**: 현재 이미지 전체가 "교체" 버튼이라 충돌.
  → **본문 클릭 = 초점**, **교체는 우상단 작은 아이콘 버튼(↻)** 으로 분리.
  (RightPanel 업로드 위젯으로도 교체 가능 — 보조 경로.)

## 4. Rationale

- **모델 A + 담긴 존**: readability를 코드 구조로 보호하면서 시각 풍부함을 늘린다.
  5장 세트면 표지·진술·혁신·마무리 4장이 이미지 잠재력을 가져 "이미지 부재" 직격.
- **클릭 초점만**: 사용자가 반복 클릭으로 즉시 try → 빠른 출시. 드래그·줌은
  정밀하나 터치 처리·구현이 무겁고 현재 불필요(YAGNI).
- **v1 패턴 재사용**: 새 뼈대 2개도 `showImage` 분기를 그대로 따라 일관·저위험.

## 5. Risks / 엣지

- **교체 동선 변경**: 전체 클릭=교체 → 우상단 ↻로 바뀌어 기존 사용자 습관과 다름.
  점선 드롭존(무이미지)은 그대로 "추가"로 동작하므로 신규 진입엔 영향 적음.
- **focal 후방호환**: 구 카드(focal 없음)는 center로 폴백. 저장 시에만 값 생성.
- **밴드 비율**: Statement/Closing 하단 밴드의 높이·여백이 텍스트 양에 따라
  답답할 수 있음 → 시각 게이트에서 텍스트 많은/적은 케이스 양쪽 확인.
- **검증 기준**: "빌드 성공"이 아니라 **브라우저 시각 확인**이 완료 기준
  (CLAUDE.md §13). statement·closing 이미지 유/무 + focal 적용을 `/render/demo/*`
  Playwright 캡처로 직접 본다.
