# 요소 미세조정 에디터 — 토큰-바운드 override 레이어

> 2026-06-08 | 설계 확정(브레인스토밍) | 구현 전
> 관련: `docs/18_card_design_system.md`(피부/뼈대), `docs/12_card_editor_content.md`(편집권한),
> `docs/07_api_data_model.md`(데이터 모델), `web/src/app/editor/[jobId]/page.tsx`, `web/src/components/cards/skin/*`

---

## 1. Problem

AI가 카드 전체를 "짠" 하고 생성한다(초벌). 하지만 막상 사용자 입맛에 안 맞을 때
손댈 방법이 없다. 현재 RightPanel의 글꼴/자간/행간 슬라이더는 **존재하지만 죽어 있다**
(`RightPanel.tsx`의 `fontFamily`/`letterSpacing`/`lineHeight`가 local `useState`일 뿐
카드에 적용되지 않음).

동시에 제약이 필요하다. 자유 캔버스(Canva형: 도형·아이콘·임의 글꼴 자유 삽입)는
doc 18이 진단한 **"섬(island) 불일치"** 문제를 정확히 재발시키고, 최우선 과제인
**가독성**을 해결하지 못하며, 데이터모델·CardRenderer·S7을 대규모 확장해야 한다.

> 멘탈 모델 확정: **"AI가 제안한 요소들에 대해서만 미세조정을 준다."**
> 새 요소 생성·삽입 없음. 자유도 없음. 초벌(AI) + 가속(사용자 미세조정).

## 2. Decision

AI 제안값 위에 얹는 **토큰-바운드 미세조정 레이어**를 추가한다.

### 2.1 강제 규칙 준수 (가장 중요)

doc 18 §2의 🔒규칙: **"스타일을 가진 primitive는 피부 컴포넌트뿐. 뼈대는 색·폰트·border 정의 금지."**

→ override는 **절대값(임의 CSS)이 아니라 토큰 어휘 안에서의 상대 선택**으로만 표현한다.
- 크기 = 역할 토큰(`--set-headline` 등) × 스텝 배수
- 색 = `--set-ink-strong | --set-ink-muted | --set-accent` 중 택1 (자유 hex 금지)
- override 적용 코드는 **피부 컴포넌트 안에만** 존재(뼈대 불변).

이로써 "토큰 = 단일 출처" 원칙과 시리즈 일관성이 깨지지 않는다.

### 2.2 3개 층

| 층 | 무엇 | 저장 위치 | 적용 범위 |
|---|---|---|---|
| **덱(deck)** | 글꼴 페어링 · 테마강조 · 배경색 | `CardDataPayload` | 전체 카드 |
| **요소(element)** | 크기·자간·굵기·정렬·색 | `Card.field_styles[fieldKey]` (신규) | 선택한 텍스트 |
| **카드(card)** | 이미지(url/focal/fit) | `Card.*` (기존) | 그 카드 |

> **글꼴은 덱 단위**(카드 단위 아님). 근거: `theme`/`bg_color`가 이미 덱 단위라 전 카드
> 동일 적용되고, 시리즈가 한 세트로 보이려면 글꼴도 전 카드 통일이어야 한다.
> 카드별로 폰트가 다르면 "불일치"가 재발한다.

### 2.3 요소 override 스펙

```ts
// web/src/types/editor.ts (신규)
export interface FieldStyle {
  size?: 'S' | 'M' | 'L' | 'XL'              // 역할 토큰 × 배수
  tracking?: number                          // letter-spacing(em), [-0.05, +0.1] 클램프
  weight?: 'regular' | 'bold'
  align?: 'left' | 'center' | 'right'
  color?: 'ink-strong' | 'ink-muted' | 'accent'   // --set-* 토큰 키만
}

export interface Card {
  // ...기존 필드...
  field_styles?: Record<string, FieldStyle>  // 신규: fieldKey → override
}

export interface CardDataPayload {
  // ...기존: cards, theme, recommended_theme_key, bg_color...
  font_pairing?: string                      // 신규(P2): 페어링 레지스트리 키
}
```

- 크기 배수(예): `{ S: 0.85, M: 1.0, L: 1.18, XL: 1.4 }` — 피부 컴포넌트가
  `fontSize: calc(var(--set-headline) * <배수>)`로 적용. (정확한 값은 구현 시 시각 튜닝)
- **모든 필드 선택적.** override 없으면 100% AI/세트 기본값.
- **"↺ AI 제안값으로 되돌리기"** = 해당 `field_styles[key]` 삭제(키 제거).

### 2.4 선택 메커니즘 (재사용)

신규 선택 시스템을 만들지 않는다. 기존 `focusedField`/`onFieldFocus`/`focused`
(`components/cards/types.ts`, `EditableText`)를 그대로 재사용.

- 캔버스에서 텍스트 클릭 → `focusedField` 설정 → RightPanel 맨 위 "선택 요소" 아코디언
  자동 오픈, 해당 fieldKey의 `field_styles` 손잡이 표시.
- 포커스 해제 → 패널은 덱/카드 단위 섹션만.

### 2.5 적용 지점 (피부 컴포넌트 5개)

텍스트 피부 컴포넌트 — `Eyebrow · Headline · Subhead · Body · Caption` — 가
`fieldStyle?: FieldStyle` prop을 받아 자기 base 스타일에 머지한다.

```
base(역할 토큰) → fieldStyle 머지 → 최종 style
```

- 뼈대(skeleton, `cards/skeletons/*`)는 `card.field_styles[key]`를 해당 피부 컴포넌트에 prop으로 넘기기만 함
  (스타일 정의 아님 → 강제 규칙 유지).
- 복합/특수 컴포넌트(CompareBars · StepFlow · BigStat 등)는 **MVP 범위 밖**(텍스트 5개만).

### 2.6 렌더·영속 경로

- **렌더 단일 소스**: 미리보기(`mode='edit'`)와 S7 PNG(`mode='render'`)가 같은 피부
  컴포넌트를 거치므로, override가 **PNG에도 자동 반영**된다(별도 렌더 경로 없음).
- **영속**: `field_styles`/`font_pairing`는 cardData JSON에 포함 → 기존 저장 흐름
  (`debouncedSave`/`saveNow` → 백엔드 → 재로드)으로 라운드트립.

### 2.7 RightPanel = 맥락 반응형 아코디언 바

승인된 구조. 아코디언은 **한 번에 하나만** 열림(기존 색상 행 패턴과 동일).

```
[▾/▸ 선택 요소: {필드명}]   ← focusedField 있을 때만, 자동 오픈
       크기 · 자간 · 굵기 · 정렬 · 색 · ↺되돌리기
[▸ 글꼴 & 스타일]           ← 덱: 글꼴 페어링(P2)
[▸ 색상]                    ← 덱: 배경/테마강조(기존 로직 이전)
[▸ 이미지]                  ← 카드: 슬롯/focal/fit(기존 로직 이전)
```

### 2.8 페이징

- **P1** (이번): 요소 override(크기·자간·굵기·정렬·색) + 패널 재구조 + 죽은 슬라이더 제거/배선.
  글꼴 페어링 섹션은 자리만(비활성/"준비 중").
- **P2** (후속): 덱 글꼴 페어링 — 큐레이트 페어링 레지스트리 + 필요 시 `--set-font-head`
  토큰 추가. 가장 무거워서 분리.

## 3. Rationale

- **가독성(최우선)을 안 망친다**: 손잡이가 토큰 스텝/범위로 묶여 레이아웃·대비를 깨기 어렵다.
- **일관성**: 글꼴 덱 단위 + 색 토큰 한정 → 시리즈 통일 유지.
- **저비용**: 선택(focusedField)·렌더(단일 컴포넌트)·저장(cardData JSON) 모두 기존 인프라 재사용.
- **fidelity 무간섭**: override는 순수 표현 레이어. `FieldValue`의 value/source/risk(근거 계약)는
  건드리지 않음(별도 `field_styles` 맵).

## 4. 영향 받는 파일 (개략)

- `web/src/app/editor/[jobId]/page.tsx` — `focusedField`·`field_styles` 업데이트 핸들러 배선,

## 6. Risks / Open Questions

- **백엔드 라운드트립**: cardData 영속 계층이 미지의 키(`field_styles`/`font_pairing`)를
  보존하는지 확인 필요. 스키마 검증이 strict면 통과 허용 추가.
- **선택 UX 충돌**: 현재 focus는 inline 텍스트 편집(contentEditable)용. "스타일 선택"과
  "텍스트 편집"이 같은 focus를 공유 — 클릭=편집+선택 동시. 구현 시 자연스러운지 검증.
- **크기 배수 시각 튜닝**: S~XL 배수는 뼈대별로 overflow 안 나는지 브라우저 검증 필요
  (CLAUDE.md/web 규칙: 완료 기준 = 시각 검증).
- **복합 컴포넌트**: BigStat/CompareBars/StepFlow는 MVP 제외 — 사용자가 그 요소 클릭 시
  "이 요소는 조정 불가" 처리 방식 결정 필요.

## 7. Out of Scope (명시적 폐기)

- 자유 캔버스 / 임의 위치 텍스트·도형·아이콘 삽입 (Canva형) — **폐기**
- 요소별 자유 글꼴 선택 (페어링 깨짐) — 글꼴은 덱 단위로 한정
- px 자유 입력 크기 / 자유 hex 색 — 토큰 어휘로 한정
- 복합/특수 피부 컴포넌트 override — 후속
