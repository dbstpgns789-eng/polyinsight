# 세트 전문화 v1 — 장식·타이포 어휘 심화 (P1)

> PolyInsight | 2026-06-07 | 카드 디자인 시스템 심화
> 상태: 설계 합의, 플랜 대기
> 상위 맥락: `project-design-system-pivot` 북극성(템플릿-세트 DB)의 첫 깊이 작업

## Problem

"콘텐츠 품질 고도화"를 시작한다. 대화로 정의를 맞춘 결과 여기서 "콘텐츠 품질"은 텍스트/가독성이 아니라
**① 시각적 풍부함(전문 제작업체 수준) + ② 사용자 제어 + ③ 다양한 세트(다양성)** 세 기둥이다.

코드로 검증한 현황 — 세 기둥 모두 "기반은 됐는데 깊이/배선이 없다":
- ① 장식 어휘 빈약(`BgMotif` 1개, Pill/TintedBox/LabeledRow 미구현), 대부분 카드가 "그라데이션 위 텍스트" → 여전히 "템플릿 티".
- ② 세트 전환·템플릿 교체·focal/zoom UI·카드 추가/삭제 미배선, 타이포 컨트롤은 UI만 있고 카드 미반영(orphan).
- ③ 세트가 **DEEP_TEAL 1개뿐**. 토큰만 교체하면 추가 가능(grep 검증)하나 `set_key` 데이터모델·선택 UI·CardFrame 배선 전무.

시각 베이스라인은 디자인 시스템 전환 + 실독자 검증("이전보다 훨씬 나아졌다")까지 왔으나, KITECH 공식 계정에
올릴 "전문 제작업체 수준"에는 못 미친다.

## Decision

**세트를 늘리기 전에, 단일 세트의 컴포넌트 어휘 깊이를 먼저 끌어올려 품질 천장을 높인다.**

핵심 통찰(박사님 동의): **세트 수 ≠ 전문성.** 컴포넌트 어휘(디자인의 "심장")가 얇으면 세트를 늘려도
"색만 다른 6벌"이 된다. 전문 수준은 색이 아니라 **컴포넌트 어휘 깊이 + 이미지 통합**에서 나온다.

따라서 첫 서브프로젝트(P1) = **장식·타이포 어휘 심화**(이미지 통합 P2는 다음). 박사님 선택:
주도 축 = "장식·타이포 깊이 먼저".

### 미감 목표
- **미니멀 프리미엄** — 절제 = 프리미엄. 장식을 *더 얹는* 게 아니라 타이포 위계·여백 리듬·미세 디테일·
  데이터 시각화를 *정제*한다.
- **기관 신뢰감** — KITECH(한국생산기술연구원) 공식 계정 발행 가능성 → 신뢰·정돈된 인상.
- **망고보드식 정보카드 중심** — 데이터 중심·가독성·시리즈 일관성.
- **열린 갈림** — 현 딥틸(어둠+골드) 심화 vs 더 밝은 미니멀 프리미엄 전환. **실행 1단계 시각 스파이크**(브라우저
  목업 2~3안 → 박사님 선택)로 해소. 텍스트로 단정하지 않는다.

## 서브프로젝트 분해 & 순서

| # | 서브프로젝트 | 상태 |
|---|---|---|
| **P1** | **세트 전문화 v1 — 장식·타이포 어휘 심화** | ← 이번 |
| P2 | 이미지 통합 심화 (focal/zoom UI + 더 많은 뼈대 이미지 존 + 세트 정체성 유지 트리트먼트) | 다음 |
| P3 | 멀티세트 기반 + 세트 전환 (set_key 데이터모델/렌더/선택 UI, 테마·세트 혼선 정리) | 후속 |
| P4 | 편집 control 확장 (템플릿 교체, 카드 추가/삭제, 타이포 orphan UI, 인라인 강조) | 후속 |
| — | 섬 12개 철거 (위생) | 병행 가능 |

각 서브프로젝트는 자체 spec → plan → 구현 사이클. 이 문서는 **P1만** 다룬다.

## P1 범위

**IN:**
1. **시각 방향 스파이크** — impeccable + 라이브 브라우저로 미감 2~3안 목업 → 박사님 선택으로 fork 확정.
2. **타이포 위계 심화** — display/headline/subhead/body/caption/eyebrow 스케일·웨이트·강조(*em*)를 정보카드급
   위계로 재정립. 토큰 확장 동반.
3. **장식 시스템 정제** — `BgMotif` 뼈대별 변주, `AccentDivider`/구분선 체계, surface/tint 깊이, 여백 리듬.
   (미니멀 — 절제된 깊이, 클러터 금지.)
4. **데이터 시각화 폴리시** — `BigStat`·`CompareBars`를 정보카드 수준으로 고도화.
5. **보류 컴포넌트 도입(전문성 기여분만)** — Pill·TintedBox·LabeledRow. YAGNI로 실제 쓰이는 것만.
6. **8뼈대 리프레시** — 갱신 어휘로 cover_v2~closing_v2 전부 재단장, 시각 게이트 통과.

**OUT (후속):**
- 이미지 통합 심화(P2), 멀티세트 배선(P3), 편집 control 확장(P4)
- 섬 12개 철거(이번엔 공존 유지), S6 프롬프트/모델/가독성(별도 트랙)

## Rationale

- **심장 우선** — `project-design-system-pivot`의 자기 경고: "심장이 빈약하면 섬 재발". 세트 양산(#3) 전에
  어휘를 두껍게 해야 다양성이 진짜 다양성이 된다.
- **미니멀 = 기관 적합** — KITECH 공식 발행 가능성은 절제·신뢰의 미감을 요구. 화려함보다 정제.
- **시각은 시각으로 결정** — 미감 fork는 텍스트 논쟁이 아니라 목업으로(CLAUDE.md §13 "이식 후 브라우저 시각
  검증", 슬라이스1·2의 "시각 게이트 PASS" 패턴 계승).
- **아키텍처 준비됨** — 뼈대는 토큰만 사용(grep 0건 검증). 어휘 심화는 컴포넌트/토큰 층에 갇혀 뼈대 파괴 없음.

## Risks / 주의

- **클러터 위험** — "장식 깊이"를 오해해 요소를 남발하면 미니멀 프리미엄과 모순. 절제 원칙을 강제규칙처럼.
- **강제규칙 유지** — 새 컴포넌트/토큰도 뼈대에 색/배경/폰트/border 직접값 0건(grep 게이트)을 깨면 안 됨.
- **S6 계약** — 가능하면 S6 불변. 신규 컴포넌트는 기존 필드 재사용 우선. 새 필드 포맷 불가피 시
  `backend/agents/s6_card_json.py` `_TEMPLATE_SPEC` 동기화(데이터모델은 자유 딕셔너리라 충격 적음).
- **섬 공존** — 신규 `_v2` 키만 건드림. 레거시 12 템플릿 무손상.
- **데모 의존 함정** — 데모는 mock이라 실제 job과 다를 수 있음(예: risk_level 부재). 시각 검증은 데모로 하되
  실 job 렌더 회귀도 확인.
