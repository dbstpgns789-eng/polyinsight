# 에디터 자잘 기능 완성 (설계)

> 2026-06-08 | 단일 카드 다운로드 · 파일명 편집 · 대시보드 프로젝트 삭제

## Problem

에디터/대시보드에 버튼만 있고 동작이 없는 미완 기능들:
- MidCanvas 하단바 다운로드 버튼 = no-op
- Topbar 파일명 = 읽기 전용(편집 미구현)
- 대시보드 프로젝트 삭제 = `TODO: DELETE /api/jobs/:id` stub
- (발견) `EditorTopBar.tsx` = 미사용 중복(라이브는 `Topbar.tsx`)

이번 범위: 위 3개 + 죽은 파일 정리. (카드 추가/삭제·복제는 제외)

## Decision

### 1. 단일 카드 다운로드 — 즉석 렌더 (A)
기존 `/image/:n`은 사전 렌더 캐시라 export 전이면 404·편집 후면 옛 이미지. 대신
**그 카드 1장을 S7로 새로 렌더**해 항상 현재 저장 상태를 정확히 반영.

- **BE**: `GET /api/cards/{job_id}/download/{card_num}` — `_playwright_render_via_url_sync([f"{WEB_BASE_URL}/render/{job}/{n}"])`로 1장 렌더 → PNG. `Content-Disposition: attachment; filename="{safe_title}_{nn}.png"`. card_data는 자동저장으로 DB에 존재.
- **FE**: MidCanvas 다운로드 버튼 onClick → 엔드포인트 blob fetch → 브라우저 다운로드. 렌더 중 비활성/스피너. demo job은 비활성(백엔드 없음).

### 2. 파일명 편집
표시 "파일명" = `jobs.title`(업로드 PDF명). 자동저장은 card_data만 PATCH하므로 별도 경로.

- **BE**: `db.update_job_title(job_id, title)` + `PATCH /api/jobs/{job_id}` body `{title}` → `{ok, title}`. 빈/공백 title 거부(400).
- **FE**: Topbar 파일명 칩 클릭 → 인라인 `<input>` → Enter/blur 저장(낙관적). 빈 값/취소(Esc)면 원복. editor가 `filename` 로컬 상태 + `onRename` 핸들러 보유.

### 3. 대시보드 프로젝트 삭제
- **BE**: `db.delete_job(job_id)` cascade(`card_images`·`card_data`·`exports`·`jobs`) + `DELETE /api/jobs/{job_id}` → 204. 없는 job은 404.
- **FE**: 대시보드 삭제 버튼(현 stub) → 확인 → DELETE → 목록에서 제거(낙관적). 실패 시 복원.

### 4. 청소
- `web/src/components/editor/EditorTopBar.tsx` 삭제(미사용 중복).

## 변경 면
- **BE**: `core/db.py`(update_job_title·delete_job) · `routers/jobs.py`(PATCH·DELETE) · `routers/export.py`(download 라우트) · 단위테스트.
- **FE**: `components/editor/MidCanvas.tsx`(다운로드) · `Topbar.tsx`(인라인 편집) · `app/editor/[jobId]/page.tsx`(handleDownloadCard·handleRename·filename 상태) · `app/dashboard/page.tsx`(삭제 배선) · `lib/api.ts`(deleteJob·renameJob·downloadCardUrl).

## Rationale
- 즉석 렌더(A): "다운로드했더니 옛 카드/404"는 나쁜 UX. 1장 비용은 작다.
- 파일명=title 단일 출처. 별도 PATCH로 자동저장 경로와 분리(혼선 방지).
- 삭제 cascade: 고아 row(card_images·exports) 방지.

## Risks
- demo job 다운로드 불가(백엔드 없음) → 버튼 비활성으로 처리.
- 즉석 렌더는 dev 서버(3000)가 떠 있어야 함(S7 전제와 동일).
- 낙관적 업데이트 실패 복원 처리 필요.

## 비목표
카드 추가/삭제/복제, 템플릿 교체, 인라인 강조, 타이포 orphan, RightPanel 세트 전환(P3).
