# User Flow
> PolyInsight v2.0 | 2025-05-05

---

## 1. 전체 흐름 개요

4개 화면으로 구성. 대시보드가 앱 루트.
업로드·내보내기는 별도 라우트 없이 오버레이 모달로 동작.

```
[대시보드 /dashboard]
    │
    ├─ "새 논문 업로드" 클릭
    │       ↓
    │  [업로드 모달 — IDLE]
    │       → 파일 선택 → FILE_SELECTED
    │       → 제출 → PROCESSING (백그라운드)
    │       → 완료 → DONE → "카드 에디터 열기"
    │       → 실패 → ERROR → 재시도 / 건너뛰기
    │
    └─ 프로젝트 카드 "수정 / 이어하기" 클릭
            ↓
       [카드 에디터 /editor/:jobId]
            → 콘텐츠 패널: 텍스트 수정
            → 프리뷰 패널: 실시간 확인
            → 디자인 패널: 레이아웃 + 이미지
            → ActionBar: 검토 완료 → "PNG 내보내기"
                    ↓
               [내보내기 모달 — PREFLIGHT]
                    → 체크리스트 확인
                    → RENDERING (Playwright)
                    → DONE → ZIP 다운로드
                    → ERROR → 재시도
```

---

## 2. 화면별 상세 흐름

### 2-1. 대시보드

**진입**: 앱 루트 (`/dashboard`)

**표시 요소**:
- 통계 카드 4개: 전체 / 완성 / 임시저장 / 처리중
- 프로젝트 그리드 (3열)
- 최근 활동 피드

**프로젝트 카드 상태별 액션**:

| 상태 | 표시 | 가능한 액션 |
|---|---|---|
| 완성 | 썸네일 5장 불투명 | 수정, 다운로드 |
| 임시저장 | 썸네일 일부 흐림 | 이어하기 |
| 처리중 | 흐림 + 진행률 바 | 없음 (자동 갱신) |
| 실패 | 흐림 + 에러 배지 | 재시도 |

---

### 2-2. 업로드 모달

**트리거**: 대시보드 "새 논문 업로드" 버튼

**상태 전이**:

```
IDLE → FILE_SELECTED → PROCESSING → DONE
                                  → ERROR
```

**IDLE**: 드래그앤드롭 영역 + 파일 선택 버튼. PDF 외 형식 업로드 시 즉시 차단.

**FILE_SELECTED**: 파일명, 크기 표시. 50 MB 초과 시 제출 전 차단.

**PROCESSING 진행 표시**:

```
✓  S1 텍스트 추출
✓  S2 섹션 파싱
⟳  S3 한국어 요약   ─┐ 병렬
⟳  S4 핵심 기여 추출─┘
○  S6 카드뉴스 JSON 생성
   (S5 없음 — KITECH 버전)
```

> 모달을 닫아도 파이프라인은 백그라운드에서 계속 실행.
> 대시보드에서 진행률 폴링으로 실시간 확인 가능.

**DONE 화면**:
- 자동 파싱된 논문 제목, 연구자 이름 확인
- 레이아웃 미리보기 (AI 추천 변형)
- 메인 CTA: **"카드 에디터 열기"** → `/editor/:jobId` 이동

**ERROR 화면**:
- 실패 스테이지 및 오류 메시지 표시
- 재시도 버튼 (실패 지점부터 재실행)
- 건너뛰기 버튼 (대시보드로 복귀)

---

### 2-3. 카드 에디터

**레이아웃**: 고정 TopBar + 3패널 + 하단 ActionBar

#### 좌측 — 콘텐츠 패널 (300px 고정)

- 탭: **웹진정보** / **카드1** / **카드2** / **카드3** / **카드4** / **카드5**
- 필드별 신뢰도 배지:
  - `CRITICAL` — 빨강: 수치 + 원문 매칭 실패. 반드시 사람 검증 필요.
  - `HIGH` — 주황: 원문 근거 있으나 해석 개입. 검토 권장.
  - `MEDIUM` — 노랑: 요약·재구성 수준. 참고용.
  - `LOW` (배지 없음): 메타데이터 등 단순 정보.
- CRITICAL 필드: 원문 출처(섹션명, 페이지 번호) 함께 표시 + **"확인 완료"** 버튼
- 필드 수정 시 즉시 `USER_VERIFIED` 상태로 전환 → 배지 색 회색 처리

#### 중앙 — 프리뷰 패널

- 미니 썸네일 스트립 5장 (상단 고정)
- 카드 번호 탭 — 경고 있으면 빨간 점 표시
- 실시간 카드 렌더링 (입력 debounce 300ms)
- 프리뷰는 Playwright 렌더링 전 CSS 프리뷰. 픽셀 동일 보장 안 됨.

#### 우측 — 디자인 패널 (200px 고정)

- 레이아웃 변형 선택: **A / B / C / D** (AI 추천 변형에 배지 표시)
- 이미지 슬롯 — 선택한 변형에 따라 슬롯 위치 자동 변경
- 이미지 미업로드 시 내보내기 차단 아님 (경고만)

#### 하단 — ActionBar

| 상태 | 버튼 표시 | 동작 |
|---|---|---|
| CRITICAL + HIGH > 0 | "수정 필요 N건" — 비활성 (회색) | 클릭 시 첫 번째 경고 필드로 스크롤 |
| CRITICAL + HIGH = 0 | "PNG 내보내기" — 활성 (네이비) | 내보내기 모달 열기 |

**자동저장**: 마지막 입력 후 5초 idle 경과 시 자동저장. TopBar에 저장 상태 표시.

---

### 2-4. 내보내기 모달

**트리거**: 카드 에디터 ActionBar "PNG 내보내기" 버튼

**상태 전이**:

```
PREFLIGHT → RENDERING → DONE
                      → ERROR
```

**PREFLIGHT 체크리스트**:

| 항목 | 통과 기준 | 차단 여부 |
|---|---|---|
| 텍스트 완성 여부 | CRITICAL/HIGH 0건 | 차단 (ActionBar에서 이미 필터) |
| 웹진 정보 입력 | 월호, 번호 입력 완료 | 경고 (차단 아님) |
| 이미지 슬롯 | 미업로드 슬롯 수 표시 | 경고 (차단 아님) |
| 출력 정보 안내 | 1080×1080 PNG × 5장 | 정보 표시 |

**RENDERING**:

```
카드 1 ████████░░  렌더링 중…
카드 2 ──────────  대기
카드 3 ──────────  대기
카드 4 ──────────  대기
카드 5 ──────────  대기
```

- 카드 1→5 순차 렌더링. 각 완료 시 즉시 표시.
- 카드당 최대 15초. 5장 전체 최대 25초.

**DONE**:
- 썸네일 5장 표시 + 개별 PNG 다운로드 링크
- **"ZIP 전체 다운로드"** — `kitech_{slug}_{YYYYMM}.zip`

**ERROR**:
- 실패한 카드 번호 + 오류 메시지 표시
- 완료된 카드 부분 ZIP 다운로드 제공
- **"실패 카드 재시도"** — 실패 카드만 재렌더링 (완료 카드 유지)

## 3. 최소 개입 경로 (Happy Path 소요 시간)

| 단계 | 담당 | 예상 시간 |
|---|---|---|
| PDF 업로드 | 사람 | ~5초 |
| S1~S6 파이프라인 | AI | ~30초 |
| 월호·번호 입력 + 연구자 확인 | 사람 | ~30초 |
| 카드 에디터 검토 (이상 없을 때) | 사람 | ~30초 |
| Playwright 렌더링 | 시스템 | ~20초 |
| **총계** | | **~2분** |

> 수치 검증이 필요한 경우 추가 시간 소요. 이 경우에도 기존 1~2주 대비 압도적으로 빠름.

## 4. 엣지 케이스 처리

| 상황 | 처리 방식 |
|---|---|
| 파이프라인 중 브라우저 닫힘 | 백그라운드 실행 유지. 재방문 시 대시보드에서 상태 확인. |
| 에디터 중 네트워크 끊김 | 자동저장 실패 시 TopBar에 "저장 실패" 표시. 재연결 후 재시도. |
| 동일 PDF 재업로드 | 별도 프로젝트로 생성. 중복 차단 없음. |
| 렌더링 중 모달 닫기 | 렌더링 계속 진행. 대시보드에서 완료 후 다운로드 가능. |
| TTL 만료 후 다운로드 시도 | "파일이 만료되었습니다" 안내 + 재렌더링 버튼 (OQ-4 결정 후 확정). |
