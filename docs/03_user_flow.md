# User Flow
> PolyInsight v2.1 | 2026-05-20

---

## 1. 전체 흐름 개요

**인증 필수 구조.** 로그인 없이 사용 불가. user_id 기반 DB 저장이 전제.

6개 화면으로 구성. 대시보드가 인증 후 앱 루트.
업로드·내보내기는 별도 라우트 없이 오버레이 모달로 동작.

```
[랜딩 /]
    │
    ├─ "무료로 시작하기" 클릭  →  [회원가입 /signup]
    └─ "로그인" 클릭          →  [로그인 /login]
                                        │
                              인증 성공 ↓
                              [대시보드 /dashboard]
                                        │
                  ┌─────────────────────┴──────────────────────┐
                  │                                            │
         "새 카드뉴스 만들기" 클릭                   프로젝트 카드 클릭
                  ↓                                            ↓
        [업로드 모달 — IDLE]                         [카드 에디터 /editor/:jobId]
                  │
                  → 파일 선택 → FILE_SELECTED
                  → 제출 → PROCESSING (모달 내 인라인)
                  → 완료 → DONE → "에디터로 이동"
                  → 실패 → ERROR → 재시도 / 닫기
                                        │
                              [카드 에디터 /editor/:jobId]
                                        │
                              "내보내기" 클릭 (CRITICAL 항목 0개일 때)
                                        ↓
                              [내보내기 모달]
                                        │
                              ZIP 다운로드 → 에디터 복귀
```

## 2. 인증 흐름

> **구현 전략 — Stub Auth**
> 실제 이메일 인증·JWT는 마지막에 구현. 현재는 `localStorage` 플래그로 전체 플로우 체험 가능하게 유지.
> 교체 범위: 폼 제출 핸들러 + 라우트 가드 2곳.

### 2-1. 회원가입 (/signup)

**레이아웃**: split-screen (좌: 브랜드 비주얼, 우: 폼)

**진입 경로**:
- 랜딩 "무료로 시작하기" 클릭
- 로그인 화면 하단 "계정이 없으신가요? 회원가입" 클릭

**폼 필드**: 이메일 / 비밀번호 / 비밀번호 확인

**완료 후**: `/dashboard` 리다이렉트

### 2-2. 로그인 (/login)

**레이아웃**: split-screen (좌: 브랜드 비주얼, 우: 폼) — 회원가입과 동일 레이아웃

**진입 경로**:
- 랜딩 헤더 "로그인" 클릭
- 회원가입 화면 하단 "이미 계정이 있으신가요? 로그인" 클릭
- 미인증 상태로 `/dashboard`, `/editor/:jobId` 직접 접근 → 로그인으로 리다이렉트

**폼 필드**: 이메일 / 비밀번호 / 비밀번호 찾기 링크

**완료 후**: 원래 요청한 URL 또는 `/dashboard` 리다이렉트

### 2-3. 인증 보호 라우트

| 라우트 | 미인증 시 동작 |
|---|---|
| `/dashboard` | `/login?redirect=/dashboard` |
| `/editor/:jobId` | `/login?redirect=/editor/:jobId` |
| `/` (랜딩) | 로그인 상태면 `/dashboard` 자동 리다이렉트 |

## 3. 화면별 상세 흐름

### 3-1. 대시보드 (/dashboard)

**진입**: 인증 후 앱 루트

**표시 요소**:
- 통계 카드 4개: 전체 / 완성 / 임시저장 / 처리중
- 프로젝트 그리드 (3열)
- 최근 활동 피드

**프로젝트 카드 상태별 액션**:

| 상태 | 표시 | 가능한 액션 |
|---|---|---|
| 완성 | 썸네일 5장 불투명 | 수정, 다운로드 |
| 임시저장 | 썸네일 일부 흐림 | 이어하기 |
| 처리중 | 흐림 + 진행률 바 | 없음 (자동 갱신) |
| 실패 | 흐림 + 에러 배지 | 재시도 |

---

### 3-2. 업로드 모달

**트리거**: 대시보드 "새 카드뉴스 만들기" 버튼

**상태 전이**:

```
IDLE → FILE_SELECTED → PROCESSING → DONE
                                  → ERROR
```

**IDLE**: 드래그앤드롭 영역 + 파일 선택 버튼. PDF 외 형식 즉시 차단.

**FILE_SELECTED**: 파일명, 크기 표시. 50 MB 초과 시 제출 전 차단.

**PROCESSING 진행 표시 (모달 내 인라인)**:

```
✓  텍스트 추출          (S1)
✓  섹션 분류            (S2)
●  핵심 내용 분석 중... (S6)  ← 현재 단계 강조
○  카드 생성            (S7)
○  저장                 (S8)
```

> 모달을 닫아도 파이프라인은 백그라운드에서 계속 실행.
> 대시보드에서 진행률 폴링으로 실시간 확인 가능.

**DONE 화면**:
- 논문 제목, 연구자 이름 확인
- "에디터로 이동" CTA → `/editor/:jobId`
- 3초 후 자동 전환

**ERROR 화면**:
- 실패 스테이지 및 오류 메시지
- 재시도 버튼 (실패 지점부터 재실행)
- 닫기 버튼 (대시보드로 복귀)

---

### 3-3. 카드 에디터 (/editor/:jobId)

**레이아웃**: TopBar + 3패널(좌·중·우) + 하단 ActionBar

**좌측 — 콘텐츠 패널 (300px 고정)**
- 탭: 웹진정보 / 카드1~5
- 필드별 신뢰도 배지: CRITICAL(빨강) / HIGH(주황) / MEDIUM(노랑) / LOW(없음)
- CRITICAL 필드: 원문 출처(섹션명, 페이지) + "확인 완료" 버튼
- 수정 시 `USER_VERIFIED` 전환 → 배지 회색 처리

**중앙 — 프리뷰 패널**
- 미니 썸네일 스트립 5장 (상단 고정)
- 페이지 단위 전환 (← → 버튼, 키보드 방향키). 스크롤로 페이지 전환 없음.
- 카드 ↔ 콘텐츠 패널 양방향 연동

**우측 — 디자인 패널 (200px 고정)**
- 테마 컬러 선택
- 기관 정보 (로고, 기관명, 부서)
- 카드 구성 목록 + 상태

**ActionBar**:

| 상태 | 버튼 | 동작 |
|---|---|---|
| CRITICAL + HIGH > 0 | "수정 필요 N건" (비활성) | 첫 경고 필드로 스크롤 |
| CRITICAL + HIGH = 0 | "PNG 내보내기" (활성) | 내보내기 모달 열기 |

**자동저장**: 마지막 입력 후 5초 idle 경과 시 자동저장. TopBar에 저장 상태 표시.

---

### 3-4. 내보내기 모달

**트리거**: ActionBar "PNG 내보내기" (CRITICAL 항목 0개일 때만 활성)

**상태 전이**:

```
PREFLIGHT → RENDERING → DONE
                      → ERROR
```

**PREFLIGHT 체크리스트**:

| 항목 | 통과 기준 | 차단 여부 |
|---|---|---|
| 텍스트 완성 여부 | CRITICAL/HIGH 0건 | 차단 (ActionBar에서 이미 필터) |
| 웹진 정보 입력 | 월호, 번호 완료 | 경고 (차단 아님) |
| 이미지 슬롯 | 미업로드 슬롯 수 표시 | 경고 (차단 아님) |

**RENDERING**: 카드 1→5 순차 렌더링. 각 완료 시 즉시 표시. 전체 최대 25초.

**DONE**: 썸네일 5장 + "ZIP 전체 다운로드" (`kitech_{slug}_{YYYYMM}.zip`)

**ERROR**: 실패 카드 재시도 (완료 카드 유지)

## 4. 헤더 상태 전환

```
비로그인:  PolyInsight  |  작동 방식  주요 기능  |  로그인   [무료로 시작하기]
                                                     ↓              ↓
                                                  /login         /signup

로그인:    PolyInsight  |  대시보드               |  [유저 아바타 ▼]  [새 카드뉴스 +]
                              ↓                                         ↓
                         /dashboard                              업로드 모달
```

- `/` 접속 시 로그인 상태면 `/dashboard` 즉시 리다이렉트

## 5. 최소 개입 경로 (Happy Path 소요 시간)

| 단계 | 담당 | 예상 시간 |
|---|---|---|
| 회원가입 / 로그인 | 사람 | ~30초 |
| PDF 업로드 | 사람 | ~5초 |
| S1~S6 파이프라인 | AI | ~30초 |
| 월호·번호 입력 + 연구자 확인 | 사람 | ~30초 |
| 카드 에디터 검토 (이상 없을 때) | 사람 | ~30초 |
| Playwright 렌더링 | 시스템 | ~20초 |
| **총계** | | **~2분 30초** |

## 6. 엣지 케이스 처리

| 상황 | 처리 방식 |
|---|---|
| 파이프라인 중 브라우저 닫힘 | 백그라운드 실행 유지. 재방문 시 대시보드에서 상태 확인. |
| 에디터 중 네트워크 끊김 | 자동저장 실패 시 TopBar에 "저장 실패" 표시. 재연결 후 재시도. |
| 동일 PDF 재업로드 | 별도 프로젝트로 생성. 중복 차단 없음. |
| 렌더링 중 모달 닫기 | 렌더링 계속 진행. 대시보드에서 완료 후 다운로드 가능. |
| 미인증 상태로 보호 라우트 접근 | 로그인 페이지 리다이렉트. 인증 완료 후 원래 URL로 복귀. |
