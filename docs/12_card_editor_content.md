# 카드 에디터 — 콘텐츠 명세

> 세 번째 화면. 업로드 모달에서 "카드 에디터 열기" 클릭 시 진입.
> 경로: `/editor/:jobId`
> 구성: 상단바 + 3패널(콘텐츠 | 미리보기 | 디자인) + 하단바 + 내보내기 모달(오버레이)

---

## 영역 1 — 상단바 (TopBar)

항상 고정 표시.

### 좌측
- 뒤로가기 버튼: `대시보드` (클릭 시 /dashboard 이동)
- 구분선
- 파일명: `{업로드한 PDF 파일명}` (미로딩 시: `카드 에디터`)

### 우측
- 저장 상태 표시 (4가지)
  - `저장 중` — 자동 저장 진행 중
  - `저장됨` — 최신 상태 저장 완료
  - `미저장` — 변경사항 있음, 저장 대기 중
  - `저장 실패` — 오류 발생
- 수동 저장 버튼: `저장`

---

## 영역 2 — 콘텐츠 패널 (좌측 패널)

카드별 텍스트 필드를 확인하고 수정하는 영역.

### 카드 탭
- 생성된 카드 수만큼 탭 표시: `01`, `02`, `03` … (카드 번호)
- 탭 클릭 시 해당 카드로 전환

### 템플릿 정보
- 레이블: `Template`
- 현재 카드의 템플릿 유형 표시
- 템플릿 종류: 표지 / 훅 / 문제제기 / 3포인트 / 비교 / 4분면 / 정의 / 흐름도 / 데이터 / 성과 / 마무리 / 브랜딩

### 필드 편집
- 현재 카드의 모든 필드를 텍스트 영역으로 표시
- 필드명은 카드 구조에 따라 다름 (예: title, body, description 등)
- 각 필드는 AI 분석 신뢰도(`confidence`) 표시: `high` / `medium` / `low`

### 위험도 표시 (필드별)
- `CRITICAL` — 수치·통계 값이 원문 대조에 실패한 필드. 출처(섹션명, 페이지) 함께 표시. 확인 완료 버튼 제공.
- `HIGH` — 높은 위험도 필드. 주의 필요.
- `MEDIUM` — 중간 위험도 필드.
- 위험도 없음 — 정상 필드.

### CRITICAL 필드 확인 흐름
- 출처 정보 표시: `출처: {섹션명} · p.{페이지번호}`
- 버튼: `확인 완료` (클릭 시 해당 필드 확인 처리됨)

---

## 영역 3 — 미리보기 패널 (중앙 패널)

### 썸네일 스트립
- 전체 카드 썸네일을 가로로 나열
- 현재 선택된 카드 강조 표시
- 썸네일 클릭 시 해당 카드로 전환

### 카드 미리보기
- 현재 선택된 카드를 실제 비율로 렌더링하여 표시 (1080×1080 기준)
- 화면 크기에 맞게 자동 스케일 조정
- 로딩 중 스피너 표시

### 하단 정보
- 현재 카드 번호 / 전체 카드 수: `{N} / {전체}`
- 현재 카드 템플릿 유형

---

## 영역 4 — 디자인 패널 (우측 패널)

카드뉴스의 시각 테마와 이미지를 설정하는 영역.

### §1 — 비주얼 테마

#### AI 추천 테마 표시
- S6가 논문 도메인을 분석해 자동 선택한 테마를 강조 표시
- 레이블: `AI 추천` 배지 표시 (추천 테마 카드에만)
- 추천 근거 툴팁: `논문 도메인 분석 결과` (호버 시 노출)

#### 테마 선택 (사용자 오버라이드)
- 선택지 4개: `Forest Light` / `Deep Dark` / `Academic Gray` / `Ivory Soft`
- 클릭 시 `user_theme` 업데이트 → 즉시 미리보기 반영
- AI 추천 테마로 되돌리기: `AI 추천으로 초기화` 버튼 (user_theme이 추천과 다를 때만 노출)
- 선택된 테마는 내보내기(S7 렌더링) 시 적용됨

#### 테마 우선순위 (구현 기준)
```
user_theme이 있으면 → user_theme 사용
user_theme이 null이면 → recommended_theme 사용
```

### §2 — 이미지 슬롯

현재 카드에 이미지를 추가하거나 교체하는 영역.

#### 이미지 없는 경우
- 안내 문구: `클릭해서 이미지 추가`
- 버튼: `이미지 선택`

#### 이미지 있는 경우
- 이미지 미리보기 표시
- 이미지 삭제 버튼: 있음 (이미지 위 호버 시 노출)
- 버튼: `이미지 교체`

#### 공통 안내
- `이미지 없이도 내보내기 가능합니다.`

## 영역 5 — 하단바 (ActionBar)

항상 고정 표시.

### 좌측 — 위험도 현황
- CRITICAL 건수가 있을 때: `CRITICAL {N}건`
- HIGH 건수가 있을 때: `HIGH {N}건`
- 모두 0건일 때: `모든 필드 검증 완료`

### 우측 — 내보내기 버튼
- CRITICAL이 0건일 때: `카드뉴스 내보내기` (활성) — HIGH 건수가 있어도 진입 가능
- CRITICAL이 1건 이상일 때: `수정 필요 {N}건` (비활성)
  - 호버 시 툴팁: `CRITICAL 필드 {N}건을 먼저 확인해주세요`

---

## 영역 6 — 내보내기 모달 (오버레이)

하단바 "PNG 내보내기" 클릭 시 진입. 상태 4개: PREFLIGHT → RENDERING → DONE | ERROR

### 모달 헤더
- 제목: `PNG 내보내기`
- 닫기 버튼: 렌더링 중에는 없음, 나머지 상태에서는 있음

### 상태 A — PREFLIGHT (내보내기 전 확인)

체크리스트 (자동 확인):
- `총 {N}장의 카드 준비 완료`
- `이미지 슬롯 {설정된 수}/{전체}장 설정` (일부 없을 경우: `일부 없음 — 내보내기 가능` 표시)
- `CRITICAL/HIGH 필드 검토 완료`

버튼: `내보내기 시작`

### 상태 B — RENDERING (렌더링 중)

- 안내: `PNG 렌더링 중`
- 보조 안내: `Playwright로 카드를 생성하고 있습니다.`
- 닫기 불가

### 상태 C — DONE (완료)

- 완료 메시지: `내보내기 완료`
- 보조 안내: `{N}장의 PNG가 준비됐습니다.`
- 버튼: `ZIP 전체 다운로드`
- 카드별 개별 다운로드: `01`, `02`, `03` … (카드 번호별 PNG)

### 상태 D — ERROR (실패)

- 오류 메시지: `내보내기 실패`
- 오류 상세: `{에러 내용}`
- 버튼: `다시 시도` (PREFLIGHT로 복귀)

- 수동 저장: 상단바 `저장` 버튼
