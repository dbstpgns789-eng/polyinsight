# 10 · PolyInsight 핵심 화면 설계
> v1.2 | 2026-05-20
> 대상: 서비스 핵심 전환 흐름 — 신규 가입 + 재방문 여정
> 설계 범위: 랜딩 → 로그인/회원가입 → 대시보드 → 업로드 → 카드 에디터 → 내보내기
> v1.1 대비 변경: 인증 필수 구조로 전환, 로그인/회원가입 화면 추가, 랜딩 CTA 동작 변경

## 0. 전체 사용자 여정

### 0-1. 신규 가입

```
[랜딩 페이지]
    │
    │  "무료로 시작하기" 클릭  → /signup
    ▼
[회원가입 페이지]  split-screen
    │
    │  이메일 + 비밀번호 입력 + 가입 완료
    ▼
[대시보드]  ← 인증 후 앱 루트
    │  "새 카드뉴스 만들기" 클릭 → 업로드 모달
    ▼
[PDF 업로드 모달]
    │  PDF 드롭 + "변환 시작" 클릭
    ▼
[처리 중 상태]  ← 모달 내 인라인 전환
    │  완료 → 3초 후 자동 전환
    ▼
[카드 에디터]
    │  "내보내기" 클릭
    ▼
[내보내기 모달]
    └─ ZIP 다운로드
```

### 0-2. 재방문 (로그인 상태)

```
[랜딩 / 접속]
    │  로그인 상태 감지 → /dashboard 자동 리다이렉트
    ▼
[대시보드]
    │  기존 프로젝트 카드 클릭 → 에디터
    │  "새 카드뉴스 만들기" 클릭 → 업로드 모달
    ▼
[카드 에디터]  →  [내보내기 모달]
```

### 0-3. 헤더 상태 전환

```
비로그인:  PolyInsight  |  작동 방식  주요 기능  |  로그인(/login)  [무료로 시작하기 → /signup]
로그인:    PolyInsight  |  대시보드               |  [유저 아바타 ▼]  [새 카드뉴스 + → 업로드모달]
```

- `/` 접속 시 로그인 상태면 `/dashboard`로 즉시 리다이렉트
- "대시보드" 링크는 헤더 좌측 nav 영역에 단독 배치

## 1. 랜딩 페이지

> 구현 상태: **구현됨, 수정 필요** (`web/src/app/page.tsx`)

### 핵심 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| CTA "무료로 시작하기" 연결 | **/signup 페이지** | 인증 필수 구조 — 업로드 전 user_id 필요 |
| 헤더 "로그인" 버튼 | **/login 페이지** | 기존 `href="#"` dead link 수정 필요 |
| 이메일 수집 섹션 (`#signup`) | **제거** | 인증 흐름이 대체. 별도 이메일 수집 불필요. |
| "회원가입 없이 바로 사용 가능" 카피 | **제거** | 인증 필수 방향과 모순 |
| 카드 스택 목업 | CSS-only, 수치 73.2% 표시 | "수치가 신뢰다" 원칙 시각화 |

### 수정 필요 항목 (`web/src/app/page.tsx`)

```
- 헤더 "로그인" href="#" → href="/login"
- 헤더·히어로 "무료로 시작하기" onClick(openModal) → href="/signup"
- hero__note "회원가입 없이 바로 사용 가능" 문구 제거
- section#signup CTA 폼 전체 삭제 (이메일 수집 섹션)
- footer 링크 "#" → 추후 실제 URL로 교체 (현재 유지)
```

### 상태 정의

| 상태 | 설명 |
|---|---|
| idle | 기본 폼 |
| loading | 제출 후 대기. 버튼 "처리 중..." + disabled |
| error | 실패. 폼 위 에러 메시지 |
| success | 성공 → /dashboard 리다이렉트 |

### 라우트 구조

```
/login   → 로그인 폼 (이메일 + 비밀번호)
/signup  → 회원가입 폼 (이메일 + 비밀번호 + 비밀번호 확인)
```

두 페이지는 동일 split-screen 레이아웃 컴포넌트를 공유. 폼과 타이틀만 교체.

### 구현 전략 — Stub Auth (현재 단계)

> 실제 이메일 인증·JWT·OAuth는 마지막 단계에서 구현.
> 지금은 플로우 체험이 가능한 최소 구현만 한다.

| 항목 | Stub 구현 | 실제 구현 (나중에) |
|---|---|---|
| 로그인 제출 | `localStorage.setItem('isLoggedIn', 'true')` → /dashboard | 백엔드 API 호출 + JWT 저장 |
| 회원가입 제출 | 동일 | 이메일 인증 + DB 저장 |
| 보호 라우트 | `localStorage.getItem('isLoggedIn')` 확인 | 토큰 유효성 검증 |
| 로그아웃 | `localStorage.removeItem('isLoggedIn')` → / | 토큰 폐기 + 서버 세션 종료 |

교체 범위: 폼 제출 핸들러 + 라우트 가드 미들웨어 2곳만 수정하면 실제 인증으로 전환 가능.

| 항목 | 결정 | 이유 |

## 5. 처리 중 상태 ← 설계 핵심 구간

### 문제 정의

업로드 완료 후 AI 파이프라인(S1→S2→S6→S7→S8) 처리까지 **30초~3분** 소요.
사용자가 이 구간에서 이탈하거나 불안해할 가능성이 가장 높다.

### 결정: 모달을 유지하며 처리 중 상태로 전환

모달을 닫지 않고 **인라인 처리 화면**으로 전환한다.
→ 사용자가 다른 곳으로 이동하지 않아도 됨.
→ 완료 시 자동으로 에디터로 전환.

```
┌─────────────────────────────────────────────┐
│                                             │ ← 모달 유지, 닫기 버튼 숨김
│  분석 중입니다                               │ ← 헤더 변경
│  kim2024_smartfarm_review.pdf               │
│                                             │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │ ← 전체 진행 바
│  ████████████████░░░░░░░░░░░░  65%         │
│                                             │
│  ✓  텍스트 추출 완료         (S1)           │
│  ✓  섹션 분류 완료           (S2)           │
│  ●  핵심 내용 분석 중...     (S6)           │ ← 현재 단계 강조
│  ○  카드 생성                (S7)           │
│  ○  저장                     (S8)           │
│                                             │
│  원문에서 수치를 확인하고 있습니다.           │ ← 단계별 설명 문구
│  잠시 기다려 주세요.                         │
│                                             │
└─────────────────────────────────────────────┘
```

신뢰도 아이콘:
- `✓ 높음` — 초록
- `⚠ 보통` — 노랑
- `✗ 낮음 / CRITICAL` — 빨강 (ActionBar 잠금 유발)

#### 4-2. Card Preview (중앙) — 핵심 설계 결정

**페이지 단위 전환 (스크롤 없음)**

```
         ◀ 이전                         다음 ▶
         
    ┌─────────────────────────────────────┐
    │                                     │
    │         [카드 렌더링 영역]           │
    │         (1080×1080 → 축소 표시)     │
    │                                     │
    │                                     │
    │                                     │
    └─────────────────────────────────────┘
    
    ○ ● ○ ○ ○   ←  썸네일 닷 네비게이터
    1   2  3  4  5
```

**전환 방식**: ◀ ▶ 버튼 클릭 또는 키보드 ← → 로 페이지 전환.
스크롤로는 페이지가 바뀌지 않음 (스크롤은 Content Panel 내부 스크롤에만 사용).

**패널 연동 규칙 (확정)**:
- 카드 페이지 전환 시 → Content Panel이 해당 카드의 섹션으로 자동 스크롤
- Content Panel에서 항목 클릭 시 → 해당 항목이 속한 카드로 미리보기 전환
- 양방향 연동. 사용자가 어느 쪽을 조작해도 반대쪽이 따라옴.

**카드 크기**: 에디터 내에서 `transform: scale(0.5)` 로 540px 표시.
실제 렌더링은 S7이 1080×1080으로 처리.

## 8. 화면 간 전환 규칙

| 출발 | 도착 | 전환 방식 | 조건 |
|------|------|-----------|------|
| 랜딩 "무료로 시작하기" 클릭 | 회원가입 페이지 | 페이지 이동 | 비로그인 |
| 랜딩 "로그인" 클릭 | 로그인 페이지 | 페이지 이동 | 비로그인 |
| 로그인 / 회원가입 완료 | 대시보드 | 페이지 이동 (리다이렉트) | 인증 성공 |
| 미인증 보호 라우트 접근 | 로그인 페이지 | 페이지 이동 (리다이렉트) | 항상 |
| 대시보드 "새 카드뉴스 만들기" | 업로드 모달 | 모달 오버레이 진입 | 항상 |
| 파일 선택 + 변환 시작 | 처리 중 상태 | 모달 내 인라인 전환 | 파일 선택됨 |
| 처리 완료 | 카드 에디터 | 자동 (3초) + 수동 버튼 | 처리 성공 |
| 처리 실패 | 업로드 모달 복귀 | 버튼 클릭 | 처리 실패 |
| 에디터 "내보내기" | 내보내기 모달 | 모달 오버레이 진입 | CRITICAL 항목 0개 |
| 내보내기 완료 | 에디터 | 버튼 클릭 또는 ✕ | 항상 |
| 에디터 ← 대시보드 | 대시보드 | 페이지 이동 | 항상 (자동저장 후) |

## 9. 설계 결정 사항 요약

| 항목 | 결정 |
|------|------|
| 인증 구조 | 필수 로그인. 비로그인 사용 없음. |
| 랜딩 CTA 연결 | /signup 페이지 (인증 후 대시보드 → 업로드 모달) |
| 로그인/회원가입 레이아웃 | split-screen (좌: 브랜드 비주얼, 우: 폼) |
| 이메일 수집 섹션 | 제거 (인증 흐름이 대체) |
| 대시보드 진입점 | 인증 후 앱 루트. 헤더 nav. `/` 접속 시 자동 리다이렉트. |
| 업로드 → 에디터 전환 | 모달 내 처리 중 화면 → 3초 후 자동 전환 |
| 패널 ↔ 미리보기 연동 | 양방향 연동 확정 |
| 카드 탐색 방식 | 페이지 단위 전환 (스크롤 없음) |
| 처리 중 이탈 | "닫아도 대시보드에서 확인 가능" 문구 + S8 저장 보장 |
